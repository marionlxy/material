1.2.2 安装SMARTDATA应用

1.2.2.1 安装MySQL

前提条件

获取mysql-5.7.32-1.el7.x86_64.rpm-bundle.tar软件包，获取方式请参考[^获取外部逐渐安装包]。

操作步骤

步骤 1 以root用户登录需要安装mysql的服务器。

步骤 2 执行以下命令，创建临时工作目录。

mkdir {工作目录}

说明

{工作目录}为创建临时工作的目录，具体创建位置由现场安装人员决定，下文中提到的{工作目录}均代指该目录，需要进行替换，不能直接执行。

步骤 3 上传MySQL安装包到{工作目录}

步骤 4 执行如下命令，进入工作目录，解压上传的压缩包文件。

cd {工作目录}
tar -xvf mysql-5.7.32-1.el7.x86_64.rpm-bundle.tar

步骤 5 执行如下命令，检测原系统中MySQL是否已存在。

xxxxxxxxxx
rpm -qa|grep -i mysql
rpm -qa|grep -i maria

若存在并且确定可以清除的情况下，执行如下命令，清除MySQL的残余文件。

xxxxxxxxxx
rpm -e $(rpm -qa|grep -i mysql) --nodeps
rpm -e $(rpm -qa|grep -i maria) --nodeps
rpm -e $(rpm -qa|grep -i postfix) --nodeps

步骤 6 执行如下命令，安装MySQL。

xxxxxxxxxx
rpm -ivh mysql-community-common-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-libs-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-libs-compat-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-embedded-compat-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-embedded-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-devel-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-embedded-devel-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-client-5.7.32-1.el7.x86_64.rpm
rpm -ivh mysql-community-server-5.7.32-1.el7.x86_64.rpm

说明

在安装过程中若出现依赖不足的错误，如下图所示：

[图片]

可先执行 yum install libaio 命令，再继续安装MySQL，或者是直接在安装依赖语句后添加 --force --nodeps 强制安装依赖。

 

步骤 7 (可选) 在安装失败时可参考如下操作，卸载MySQL。

  • a. 执行如下命令，查看安装的MySQL的程序。

    xxxxxxxxxx
    rpm -qa|grep mysql

  • b. 执行如下命令，卸载安装的MySQL的程序。

    xxxxxxxxxx
    rpm -ev 程序名1 程序名2 --nodeps

  • c. 执行如下命令，查看所有关于MySQL的目录。

    xxxxxxxxxx
    find / -name mysql

  • d. 删除所有MySQL的目录

    xxxxxxxxxx
    rm -r xxx

步骤 8 执行如下命令，启动MySQL。

xxxxxxxxxx
systemctl start mysqld

说明

重启过程中若出现如下错误，可参考对应解决办法

  • 错误1：/usr/sbin/mysqld: error while loading shared libraries: libnuma.so.1: cannot open shared object file: No such file or directory

    • 解决办法： 执行yum -y install numactl 命令

  • 错误2：--initialize specified but the data directory has files in it. Aborting.

    • 解决办法：执行 rm -rf /var/lib/mysql 命令

    注意

    rm -rf 命令为危险操作，误操作的后面是丢失全部数据，必须得到客户的同意后才可操作。

步骤 9 启动成功后，修改MySQL的root账目密码。

  • a. 执行如下命令，查看数据库的初始密码。

    grep "temporary password" /var/log/mysqld.log

  • b. 若找不到初始密码可删除/var/lib目录下的mysql文件夹，再重启MySQL服务。

    xxxxxxxxxx
    rm -rf /var/lib/mysql
    systemctl restart mysqld

注意

rm -rf 命令为危险操作，误操作的后面是丢失全部数据，必须得到客户的同意后才可操作。

  • c. 根据初始密码登录MySQL服务。

    mysql -uroot -p

    #输入过的的初始密码

  • d. (可选) 修改root密码。

    登录MySQL，执行如下命令修改密码。

    alter user 'root'@'localhost' identified by '新密码'

  • e. 退出，使用新密码重新登录。

步骤 10 (可选) 给账户添加远程操作权限。

grant all on *.* to root@'%' identified by '新密码' with grant option;

步骤 11 将MySQL的初始数据导入到数据库中。

  • a. (可选) 登录数据库，执行如下命令，创建一个新的数据库用户，并将账户赋予连接权限。

    create user 'sdata'@'localhost' identified by '密码';
    grant all on *.* to 'sdata'@'%' identified by '密码' with grant option;

  • b. 执行如下命令，新建数据库database，并将其使用权限赋予新增用户；

    xxxxxxxxxx
    create database sdata;
    grant select,insert,update,delete,drop,create,REFERENCES on sdata.* to sdata;
    FLUSH PRIVILEGES;

  • c. 将数据导入到数据库中。

    • 将得到的sql文件上传到服务器指定的文件夹暂存。

    • 登录数据库，切换databfase。

      use sdata

    • 用 source 命令进行刷库。

      例如： `source {工作目录}/xxx.sql

1.2.2.3 安装SMARDATA和Tomcat

1.2.2.3.1 前提条件

步骤 1 以root与农户登录需要安装的SMARDATA服务器。

步骤 2 检查防火墙是否关闭。

步骤 3 关闭SELinux。

1.2.2.3.2 操作步骤

步骤 1 创建临时工作目录。

xxxxxxxxxx
mkdir {工作目录}

说明

{工作目录}为创建临时工作目录，具体创建位置由现场安装人员决定，下文中提到的{工作目录}均代指该目录，需要进行替换，不能直接执行。

步骤 2 上传安装包到{工作目录}

步骤 3 执行如下命令，新建操作系统用户。

xxxxxxxxxx
useradd -d {用户目录} -s /bin/bash -m {用户名}

说明

  • -d 主目录名称为{用户目录}，{用户目录}为示例，需要根据现场环境修改，不允许直接使用，下文中的{用户目录}均代指该用户目录。
  • -s shell类型，指定用户的登录shell，推荐使用/bin/bash。
  • -m 用户名，{用户名}为示例，需要根据现场环境进行修改，下文中{用户名}均代指该用户名。

步骤 4 设置用户密码。

xxxxxxxxxx
passwd {用户名}

步骤 5 将安装包拷贝到用户目录。

xxxxxxxxxx
cp {工作目录} /SMARDATA_INSTALL_.zip* {用户目录}

步骤 6 执行如下命令，修改上传源码包的用户权限。

xxxxxxxxxx
chown -R {用户名} : {用户名}{用户目录}
xxxxxxxxxx
#示例
chown -R sdata:sdata /home/sdata

步骤 7 执行 su- {用户名}，切换到新用户，解压源码压缩包到${HOME}目录下。

xxxxxxxxxx
#示例
su - sdata
tar xvf xfyq_2021_09_08.tar

步骤 8 执行如下命令，打开ombashrc文件。

xxxxxxxxxx
vi ombashrc

步骤 9 修改ombashrc文件，将元数据库信息写入。

xxxxxxxxxx
export JDBC_URL="jdbc:mysql://[ ip ]:[ port ]/[ database ]?useUnicode=true&characterEncoding=utf-8&allowMultiQueries=true&useSSL=false"
exportJDBC_USERNAME= sdata
exportJDBC_PASSWORD= ******

说明

  • ip、port和database，请根据实际环境替换，替换包含‘[ ]’方括号。

  • export JDBC_USERNAME和export JDBC_PASSWORD为安装MySQL步骤11中创建的用户名和加密后的密码，请根据实际环境替换。

  • 密码加密方法：

    xxxxxxxxxx
    cd {用户目录}/tools/encrypt
    #执行脚本
    ./encrypt.sh
    #输入明码密码

步骤 10 执行如下命令，保存并退出。

xxxxxxxxxx
:wd

步骤 11 执行如下命令，更新ombashrc文件。

xxxxxxxxxx
cd {用户目录}
source ~/ombashrc
vi~/.bash_profile
#在文件中添加一行：
source ~/ombashrc

步骤 12 按键盘上Esc键返回vi编辑器的命令模式，执行如下命令保存退出。

xxxxxxxxxx
:wq

步骤 13

步骤 14 执行如下命令，给脚本赋权。

xxxxxxxxxx
cd {用户目录}
chmod +x tomcat/bin/.sh*
chmod +x jdk/jre/bin/
chmod +x jdk/bin/**

步骤 15 将license文件sdata.lic放到${HOME}/tomcat/webapps目录下。

步骤 16 以 root 用户执行 lsof-i: 端口号 命令，查看tomcat/conf目录下server.xml文件中各 端口是否需要修改，未被占用则不用修改。 建议http端口号使用 18080 ，https端口号使用 18443 ，可以根据实际情况调整。

步骤 17 切换至{用户名}用户，执行tomcat/bin/目录下的start-om.sh文件。

xxxxxxxxxx
su - xfyq
./start-om.sh

步骤 18 查看tomcat/logs目录下的启动日志，启动无报错即可。

说明

如果启动失败，可通过以下命令查看错误日志。

xxxxxxxxxx
tail-ftomcat/logs/catalina.out //tomcat启动日志
tail-ftomcat/logs/sdata/sdata.log //系统日志

步骤 19 执行如下命令，查看tomcat是否启动。

xxxxxxxxxx
ps-ef|grep sdata|grep tomcat
